    Keepalived简介 #

    Keepalive官网：https://www.keepalived.org

    Keepalived is a routing software written in C. The main goal of this project is to provide simple and robust facilities for loadbalancing and high-availability to Linux system and Linux based infrastructures. Loadbalancing framework relies on well-known and widely used Linux Virtual Server (IPVS) kernel module providing Layer4 loadbalancing. Keepalived implements a set of checkers to dynamically and adaptively maintain and manage loadbalanced server pool according their health. On the other hand high-availability is achieved by VRRP protocol. VRRP is a fundamental brick for router failover. In addition, Keepalived implements a set of hooks to the VRRP finite state machine providing low-level and high-speed protocol interactions. In order to offer fastest network failure detection, Keepalived implements BFD protocol. VRRP state transition can take into account BFD hint to drive fast state transition. Keepalived frameworks can be used independently or all together to provide resilient infrastructures.

    Keepalived is free software; you can redistribute it and/or modify it under the terms of the GNU General Public License as published by the Free Software Foundation; either version 2 of the License, or (at your option) any later version.

    Keepalive是一个用C语言编写的路由软件。该项目的主要目标是 为 Linux 提供简单而强大的功能负载平衡和高可用性 基于系统和 Linux 的基础架构。负载均衡框架依赖于众所周知的 以及广泛使用的 Linux Virtual Server（IPVS） 内核模块，提供第 4 层负载平衡。Keepalived实现了一组检查器，用于根据负载平衡服务器池的运行状况动态地、自适应地维护和管理负载均衡服务器池。另一方面，VRRP协议实现了高可用性。VRRP是路由器故障切换的基础组件。此外，Keepalived实现了一组到VRRP有限状态机的挂钩，提供低级和高速的协议交互。为了提供最快的网络故障检测，Keepalived实现了BFD协议。VRRP状态转换可以考虑BFD提示来驱动快速状态转换。Keepalived框架可以单独使用，也可以一起使用，以提供有弹性的基础设施。

    Keepalived是免费软件；您可以根据自由软件基金会发布的GNU通用公共许可条款重新分发和/或修改它；许可证的第2版，或（由您选择）任何更高版本。

    VRRP：Virtual Router Redundancy Protocol（虚拟路由冗余协议）

    Keepalived安装 #

    注意：需要在所有节点（主、备）都进行安装

    yum方式安装 #

    直接通过yum安装软件包

    yum install keepalived

    启动服务器

    systemctl start keepalived.service && systemctl enable keepalived.service

    源码方式安装 #

    创建并进入安装目录

    mkdir -p /opt/installed && cd /opt/installed

    下载Keepalived安装包到/opt/installed/

    wget https://www.keepalived.org/software/keepalived-2.2.8.tar.gz -P /opt/installed/

    将Keepalived安装包解压到/opt/installed目录下

    tar -zxvf https://www.keepalived.org/software/keepalived-2.2.8.tar.gz -C /opt/installed

    进入Keepalived安装包目录

    cd /opt/installed/keepalived-2.2.8

    安装依赖组件

    yum -y install gcc openssl-devel net-snmp-devel libnl libnl-devel libnl3 libnl3-devel

    如果未安装libnl和libnl-devel，有可能在进行安装配置时产生如下报错：

    *** WARNING - this build will not support IPVS with IPv6. Please install libnl/libnl-3 dev libraries to support IPv4.

    进行Keepalived安装配置

    ./configure --prefix=/usr/local/keepalived --sysconf=/etc

    --prefix=/usr/local/keepalived表示将Keepalived安装在/usr/local/keepalived目录下。

    --sysconf=/etc表示将Keepalived的配置文件放在/etc目录下。

    安装Keepalived

    make && make install

    Keepalived配置文件 #

    备份配置文件

    cp /etc/keepalived/keepalived.conf.sample /etc/keepalived/keepalived.conf

    修改配置文件/etc/keepalived/keepalived.conf

     ! Configuration File for keepalived
 
 global_defs {
   # 配置通知邮箱，当Keepalived发生变化时会发送邮件
   notification_email {
     acassen@firewall.loc
     failover@firewall.loc
     sysadmin@firewall.loc
   }
   notification_email_from Alexandre.Cassen@firewall.loc
   smtp_server 192.168.1.100
   smtp_connect_timeout 30
   # 路由ID是当前安装Keepalived节点主机的全局唯一标识符，不同的主机配置不同的路由ID
   router_id keepalived_node_01
   vrrp_skip_check_adv_addr
   vrrp_strict
   vrrp_garp_interval 0
   vrrp_gna_interval 0
}
 
 # 创建名为“VI_1”的VRRP实例
 vrrp_instance VI_1 {
    # 设置当前实例的状态，MASTER为主节点，BACKUP为备份节点
    state MASTER
    # 设置当前实例绑定的网卡
    interface eth0
    # 设置虚拟路由ID，主、备节点的虚拟路由ID必须相同
    virtual_router_id 51
    # 设置优先级/权重，对于BACKUP节点来说，当MASTER节点DOWN掉后，谁的优先级高，谁将会被选举为MASTER节点
    priority 100
    # 设置主备之间心跳检测的时间间隔，默认是1秒
    advert_int 1
    # 设置认证密码，防止非法节点的接入
    authentication {
        auth_type PASS
        auth_pass 123456
    }
    # 虚拟IP，即VIP，一般情况下使用一个即可
    virtual_ipaddress {
        192.168.1.10
        # 192.168.1.11
        # 192.168.1.12
    }
}

    其他配置用不到的删除即可。

    Keepalived服务管理 #

    启动Keepalived服务并设置开机启动

    systemctl start keepalived.service && systemctl enable keepalived.service

    Keepalived的服务文件路径：/usr/lib/systemd/system/keepalived.service

    Keepalived服务检查

    systemctl status keepalived.service

    停止Keepalived服务

    systemctl stop keepalived.service

    Keepalived实现应用服务高可用 #

    虽然Keepalived可以实现主机间的高可用，但如果需要实现应用级别的高可用，则需要借助脚本来实现，下面以实现Nginx服务的高可用为例。

    创建检测Nginx服务脚本/etc/keepalived/chech_nginx_alive.sh

    #! /bin/bash
result=`ps -C b nginx --no-header | wc -l`
# 判断Nginx服务是否正常，如果异常则尝试重启服务
if [ $result -eq 0 ]; then
    systemctl restart nginx
    # 等待3秒
    sleep 3
    # 如果没启动成功，则停止keepalived，使VIP与备用机IP进行绑定
    if [ `ps -C nginx --no-header | wc -l` -eq 0 ]; then
        killall keepalived
    fi
fi

    增加脚本的执行权限

    chmod +x /etc/keepalived/chech_nginx_alive.sh

    在Keepalived的配置文件/etc/keepalived/keepalived.conf中添加检测Nginx服务的脚本

     ! Configuration File for keepalived
 
 global_defs {
   notification_email {
     acassen@firewall.loc
     failover@firewall.loc
     sysadmin@firewall.loc
   }
   notification_email_from Alexandre.Cassen@firewall.loc
   smtp_server 192.168.1.100
   smtp_connect_timeout 30
   router_id keepalived_node_01
   vrrp_skip_check_adv_addr
   vrrp_strict
   vrrp_garp_interval 0
   vrrp_gna_interval 0
}
 
 # 添加(创建)检测Nginx服务的脚本
 vrrp_script check_nginx_alive {
     script "/etc/keepalived/chech_nginx_alive.sh"
     # 每隔5秒运行一次脚本
     interval 5
     # 如果脚本运行成功，则权重+10
     weight 10
 }
 
 vrrp_instance VI_1 {
    state MASTER
    interface eth0
    virtual_router_id 51
    priority 100
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass 123456
    }
    
    # 调用检测Nginx服务的脚本
    track_script {
        check_nginx_alive
    }
    
    virtual_ipaddress {
        192.168.1.10
    }
}

    重启Keepalived服务

    systemctl restart keepalived.service

    Keepalived高可用模式 #

    Keepalived的高可用模式分为两种：

    • 主备模式：平时只有主节点工作，备节点闲置，只有当主节点无法访问时备节点才开始工作。主备模式通常使用一个VIP与主节点和备节点绑定。缺点是由于备节点平时不工作，会造成资源的浪费。
    • 双主模式：两个节点都设置为主节点，且两个节点同时工作，两个节点即是主节点也是另外一个节点的备节点。双主模式通常使用两个VIP分别绑定两个主机，且这两个主机又分别设置为对方的备节点，通过DNS轮询的方式，使两个主节点同时工作。

    Keepalived配置双主模式 #

    主机1（keepalived_node_01）配置示例

     ! Configuration File for keepalived
 
 global_defs {
   router_id keepalived_node_01
}
 
 vrrp_instance VI_1 {
    state MASTER
    interface eth0
    virtual_router_id 51
    priority 100
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass 123456
    }

    virtual_ipaddress {
        192.168.1.11
    }
}

 vrrp_instance VI_2 {
    state BACKUP
    interface eth0
    virtual_router_id 52
    priority 50
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass 123456
    }

    virtual_ipaddress {
        192.168.1.12
    }
}
