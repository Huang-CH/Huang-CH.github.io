    主机规划 #

    Ceph-01

    • eth0：10.10.10.101/24
    • eth1：192.168.1.101/24

    Ceph-02

    • eth0：10.10.10.102/24
    • eth1：192.168.1.102/24

    Ceph-03

    • eth0：10.10.10.103/24
    • eth1：192.168.1.103/24

    操作系统：OpenEuler 22.03 LTS

    基础配置 #

    修改主机名称

    hostnamectl set-hostname Ceph-0X

    修改IP地址

    vim /etc/sysconfig/network-scripts/ifcfg-ethX

    关闭防火墙

    systemctl stop firewalld.service && systemctl disable firewalld.service

    关闭SELinux

    sed -i 's/SELINUX=enforcing/SELINUX=disabled/' /etc/selinux/config

    修改/etc/hosts文件

    10.10.10.101   Ceph-01
10.10.10.102   Ceph-02
10.10.10.103   Ceph-03

192.168.1.101   Ceph-01-Cluster
192.168.1.102   Ceph-02-Cluster
192.168.1.103   Ceph-03-Cluster

    安装时间同步服务

    yum -y install chrony

    同步时间

    chronyc sources -v

    安装Docker #

    在/opt/目录下新建installer目录

    mkdir /opt/installer

    进入/opt/installer目录

    cd /opt/installer

    卸载系统自带的Docker

    yum -y remove docker-*

    下载Docker源码包

    wget -P /opt/installer/ https://download.docker.com/linux/static/stable/x86_64/docker-26.1.3.tgz

    解压Docker源码包

    tar -zxvf docker-26.1.3.tgz

    将/opt/installer/docker/目录中的所有文件拷贝到/usr/bin/目录中

    cp /opt/installer/docker/* /usr/bin/

    将Docker注册为Service服务

    vim /etc/systemd/system/docker.service

    [Unit]
Description=Docker Application Container Engine
Documentation=https://docs.docker.com
After=network-online.target firewalld.service
Wants=network-online.target
 
[Service]
Type=notify
ExecStart=/usr/bin/dockerd
ExecReload=/bin/kill -s HUP $MAINPID
TimeoutSec=0
RestartSec=2
Restart=always
 
[Install]
WantedBy=multi-user.target

    systemctl daemon-reload

    systemctl start docker.service && systemctl enable docker.service

    安装Cephadm #

    指定cephadm版本

    CEPH_RELEASE=17.2.7

    下载指定的cephadm版本

    curl --silent --remote-name --location https://download.ceph.com/rpm-${CEPH_RELEASE}/el9/noarch/cephadm

    修改cephadm权限

    chmod +x /opt/installer/cephadm

    将cephadm拷贝到/usr/sbin/目录中

    cp /opt/installer/cephadm /usr/sbin/

    检查环境是否满足cephadm的安装条件

    cephadm check-host

    安装MON节点 #

    离线环境下需要提前部署好Harbor，并且将Docker镜像通过 docker load -i <镜像名称> 命令导入所有主机。

    修改/usr/sbin/cephadm中镜像的下载地址

    在Ceph-01上安装mon服务

    cephadm bootstrap --mon-ip 10.10.10.101 --cluster-network 192.168.1.0/24 --initial-dashboard-user admin --initial-dashboard-password admin

    离线环境下，可以通过联网环境打包所有镜像

    docker save -o <output tar file> <image name>:<version>

    在离线环境下导入镜像

    docker load -i <output tar file>

    删除集群命令

    cephadm rm-cluster --fsid <fsid> --force

    在Ceph-01主机上向其他主机添加Ceph公钥

    ssh-copy-id -f -i /etc/ceph/ceph.pub Ceph-02
ssh-copy-id -f -i /etc/ceph/ceph.pub Ceph-03

    也可以使用以下命令操作

    for i in {2..5};do ssh-copy-id -f -i /etc/ceph/ceph.pub ceph-0$;done

    添加主机到Ceph集群

    cephadm shell

    ceph orch host add Ceph-02 10.10.10.102 --labels=mon,mgr
ceph orch host add Ceph-03 10.10.10.103 --labels=mon,mgr

    查看Ceph集群的主机节点

    ceph orch host ls

    给节点添加、删除标签

    # 给Ceph-01、Ceph-0添加 _admin 标签
ceph orch host label add Ceph-02 _admin
ceph orch host label add Ceph-03 _admin

# 给Ceph-01、Ceph-0添加 mon 标签
ceph orch host label add Ceph-02 mon
ceph orch host label add Ceph-03 mon

# 给Ceph-01、Ceph-0添加 mgr 标签
ceph orch host label add Ceph-02 mgr
ceph orch host label add Ceph-03 mgr

    _admin标签在默认情况下应用于存储集群中的bootstrapped主机，client.admin密钥被分发到该主机，将这个标签添加到其他主机后，其他主机的/etc/ceph下也将有client.admin密钥。

    指定mon节点个数

    ceph orch apply mon --placement="3 label:mon"

    添加OSD节点 #

    修改osd最小数量为2，默认为3（可选）

    ceph config set global osd_pool_default_size 2

    在osd节点初始化osd，将指定的磁盘格式化为无分区的原始磁盘

    blkdiscard /dev/sdb

    cephadm shell ceph orch device zap Ceph-01 /dev/sdb

    添加osd

    cephadm shell

    ceph orch daemon add osd Ceph-01:/dev/sdb

    其他硬盘如sdc、sdd、sde等，使用上述命令一次类推

    监控创建状态

    watch -n 1 ceph -s

    创建存储池 #

    创建存储池命令

    ceph osd pool create <pool_name> <pg_num> <pgp_num> <pool_type>

    <pool_name>是所创建的存储池的名称；

    <pg_num>是存储池的PG数量；

    <pgp_num>是存储池的PG原始副本数（通常与<pg_num>相同）；

    <pool_type>是存储池的类型，可以是replicated（副本池）或者erasure-coded（纠删码池）。

    创建存储池

    # 固态硬盘存储池
ceph osd pool create ssdpool 4096 4096
# 机械硬盘存储池
ceph osd pool create hddpool 4096 4096

    删除存储池，删除池时需要输入两遍池名称

    ceph osd pool rm 池名称 池名称 --yes-i-really-really-mean-it

    创建crush规则

    ceph osd crush rule create-replicated hdd default host hdd

    将crush规则应用到pool上

    ceph osd pool set HDD-Pool crush_rule hdd

    安装Ceph客户端 #

    在客户端主机（10.10.1.200）上安装ceph-common

    yum -y install ceph-common

    在MON主机（10.10.10.101）上将ceph.conf和ceph.client.admin.keyring拷贝至客户端主机（10.10.10.200）

    scp /etc/ceph/ceph.conf root@10.10.10.200:/etc/ceph/

    scp /etc/ceph/ceph.client.admin.keyring root@10.10.10.200:/etc/ceph/

    在客户端主机上查看Ceph集群信息

    ceph -s

    使用Ceph块存储 #

    块存储接口可以像使用本地磁盘一样来使用Ceph提供的块存储设备。

    使用Ceph块存储设备前，需要先创建一个RBD池。再创建RBD池之后，对存储进行定义，并创建属于该池的块存储设备。块存储设备创建完成后，就可以在Ceph客户端进行映射和挂载了。

    RBD创建与挂载 #

    创建RBD Pool

    ceph osd pool create rbd 512

    其中，512是放置组（Placement Group）的数量，放置组数量的计算公式为PGs = ( OSDs * 100 ) / pool size，其中pool size是复制池的副本数量，或用于删除代码池的K+M总和（ceph osd erasure-code-profile的返回值），其结果应舍入到接近2的指数。

    设置每个OSD中最大放置组限制数为1000

    ceph config set mon mon_max_pg_per_osd 1000

    设置RBD Pool自动扩展

    ceph osd pool set <POOL_NAME> pg_autoscale_mode on

    若要删除RBD Pool，必须设置--mon-allow-pool-delete=true参数，否则无法删除

    ceph tell mon.\* injectargs '--mon-allow-pool-delete=true'

    删除RBD Pool，注意，池名字需要输入两遍

    ceph osd pool delete <RBD_POOL_NAME> <RBD_POOL_NAME> --yes-i-really-mean-it

    在RBD Pool中创建RBD设备

    rbd create <RBD_POOL_NAME>/<RBD_NAME> --size=10T

    查看创建的RBD设备

    rbd info <RBD_POOL_NAME>/<RBD_NAME>

    映射RBD设备

    rbd map <RBD_POOL_NAME>/<RBD_NAME>

    查看RBD设备映射后的块设备名称

    rbd showmapped

    格式化块设备

    mkfs.xfs /dev/<BLOCK_DEVICE>

    创建挂载目录

    mkdir /mnt/rbd

    挂载设备

    mount /dev/<BLOCK_DEVICE> /mnt/rbd

    写入测试数据

    dd if=/dev/zero of=/mnt/rbd/testfile bs=1M count=1000

    在/etc/ceph/rbdmap中添加映射信息

    <RBD_POOL_NAME>/<RBD_NAME> id=admin,keyring=/etc/ceph/ceph.client.admin.keyring

    设置rbdmap开机自动启动

    systemctl restart rbdmap && systemctl enable rbdmap

    查看设备UUID

    blkid

    在/etc/fstab中添加RBD设备挂载信息

    UUID="30a007c3-2369-498f-9ee2-87571dbc2e8b" /mnt/rdb xfs noauto 0 0

    noauto表示等rbdmap服务启动后再执行挂载操作

    RBD快照 #

    Ceph为RBD提供了快照功能，当RBD数据出现问题或者误删除时，可以通过回放操作进行状态恢复。

    假设，RBD Pool为rbd，RBD设备为test，创建名为test_2024-06-12T15:51:01.211+08:00的快照

    rbd snap create rbd/test@test_2024-06-12T15:51:01.211+08:00

    注意，快照是只读设备，可以通过以下操作进行验证

    映射快照设备

    rbd map rbd/test@test_2024-06-12T15:51:01.211+08:00

    验证

    blockdev --getor /dev/rbd0

    blockdev --getor /dev/rbd1

    输出数值为0表示可读写，为1表示只读。

    使用快照还原RBD时，需要先取消映射，否则会报错

    umount /mnt/data

    rbd unmap rbd/test

    使用快照还原RBD

    rbd snap rollback rbd/test@test_2024-06-12T15:51:01.211+08:00

    重新映射

    rbd map rbd/test

    重新挂载

    mount /dev/rbd0 /mnt/data

    如果想要删除快照，可以使用以下命令

    rbd snap purge rbd/test

    RBD数据的导入导出 #

    RBD Image数据的导入导出常用于RBD块设备的简单备份与恢复。

    RBD Image数据导出

    rbd export rbd/test rbd_test.dat

    RBD Image数据导入

    rbd import ./rbd_test.dat rbd/test

    使用Ceph文件存储 #

    在Ceph集群中，使用文件存储必须有元数据服务器，用它来管理文件索引等信息。

    在生产环境下，为了提高文件存储的访问性能，建议部署多个元数据服务器，并且元数据服务器节点尽量不要和其他角色的节点复用。

    部署MDS #

    在Ceph Dashboard的Cluster > Services中添加MDS服务。

    创建元数据和数据池 #

    创建CephFS元数据池

    ceph osd pool create cephfs_metadata

    创建CephFS数据池

    ceph osd pool create cephfs_data

    分别为元数据池和数据池创建文件系统

    ceph fs new cephfs cephfs_metadata cephfs_data

    使用CephFS #

    创建ceph-secrect文件

    ceph auth get-key client.admin > ceph_secret

    创建挂载目录

    mkdir /mnt/cephfs

    挂载CephFS

    mount -t ceph 10.10.10.101:6789:/ /mnt/cephfs -o name=admin,secretfile=/root/ceph_secret

    写入测试数据

    dd if=/dev/zero of=/mnt/cephfs/testfile bs=1M count=1000
