    Git的安装 #

    官方网站： Git (git-scm.com)

    Fedora

    dnf install git

    Debian/Ubuntu

    add-apt-repository ppa:git-core/ppa
apt update -y
apt install -y git

    Windows

    下载地址： Git - Downloading Package (git-scm.com)

    Git的配置文件 #

    • 当前项目配置文件路径：/.git/config，使用git config --local命令进行配置，配置文件仅对当前项目（目录生效）
    • 全局配置文件路径：~/.gitconfig，使用git config --global命令进行配置，配置文件对当前用户生效
    • 系统配置文件路径：/etc/.gitconfig，使用git config --system命令进行配置，配置文件对系统下的所有用户生效

    配置文件的优先级：当前项目配置文件->全局配置文件->系统配置文件

    应用场景：

    • 配置用户信息

    Git的基本操作 #

    要让Git对一个目录进行版本控制，首先需要在Git Bash（或其他命令行工具）中进入目录

    执行初始化命令

    git init

    查看工作区中文件的管理状态

    git status

    在工作区（目录或文件夹）中，新增或修改过的文件都会是红色

    让Git忽略项目中的一些文件

    在项目中新建文件.gitigonre，在.gitigonre文件中添加想要被Git忽略的文件名称或文件夹名称（文件名支持通配符）

    vim .gitigonre

# 添加想要被Git忽略的文件名称或文件夹名称
.gitigonre
files/
*.sql
!a.sh

    将文件提交到暂存区

    # 管理指定文件
git add <文件名称>

# 管理全部文件
git add .

    提交到暂存区文件会由红色变成绿色

    配置个人信息

    在首次提交版本前，需要配置用户名称和电子邮箱，首次配置完成后，再次提交就无需再次配置了

    git config --global user.email "you@examole.com"
git config --global user.name "Your Name"

    提交到版本库

    git commit -m '描述信息'

    给版本打标签

    git tag -a 标签名称 -m '描述信息'

    查看版本记录

    git log

    以图形化的方式显示版本记录

    git log --graph

    查看引用日志

    查看所有分支的所有操作记录，包括已经被删除的 commit 记录和 reset 的操作，配合回滚版本命令，相当于有了“后悔药”

    git relog

    回滚版本

    # 从版本库回滚到暂存区（绿色）
git reset --soft <版本号>

# 将文件从暂存区（绿色）回滚到工作区（红色）
git reset HEAD <文件名称>

# 将工作区中已修改的文件（红色）回滚到未修改的状态（白色）
git checkout <文件名称>

# 从版本库直接回滚到工作区（红色）
git reset --mix <版本号>

# 从版本库直接回滚到工作区（白色）
git reset --hard <版本号>

    Git的分支 #

    分支可以给开发者提供多个开发环境，这样就可以将开发工作从主线分离出来，以避免影响主线的功能。

    查看目前所处的分支

    git branch

    创建分支

    git branch <分支名称>

    切换分支

    git checkout <分支名称>

    通过如下命令可以创建并切换到dev分支：

    git checkout -b dev

    合并分支

    要想将其他分支合并到master上，需要切换到master上进行操作

    git merge <分支名称>

    将已经合并到master的分支删除

    git branch -d <分支名称>

    Git的工作流 #

    使用Git进行开发，应至少拥有2个分支，一个master分支，一个dev分支。master分支用于发布正式版本，其余的开发工作都应在dev分支进行。

    将项目推送到Github #

    添加远程仓库地址

    git remote add origin https://github.com/xxx/xxx.git

    origin为 https://github.com/xxx/xxx.git 地址的别名，将 用户名和密码拼接到连接 https://用户名:密码@github.com/xxx/xxx.git中，可以实现免密码登录

    将项目推送到Github

    # 将master分支推送到github
git push -u origin master

# 将dev分支推送到github
git push -u origin dev

    -u表示指定默认仓库地址，首次使用git push -u指定默认仓库地址后，下次推送可不加任何参数，而直接使用git push，就可以将项目推送到origin master或origin dev

    将项目从Github中拉取到本地 #

    将Github中的项目克隆到本地

    适用于首次获取项目

    git clone https://github.com/xxx/xxx.git

    注意，本地开发时要使用dev分支，同时根据情况选择要不要将master分支合并到dev分支

    切换到dev分支

    git checkout dev

    将master分支合并到dev分支

    git merge master

    从Github仓库拉取代码

    适用于本地已有项目，需要从Github获取dev分支更新的代码

    git pull origin dev

    合并提交记录 #

    当提交记录过多或部分提交记录不被关心时，可以将这些提交记录进行合并，使提交记录变得更加简洁

    注意：尽量不要合并已经提交到版本库的记录

    git rebase -i <commit hash值>

    快速解决冲突 #

    安装Beyond Compare软件

    在Git中进行如下配置

    git config --local merge.tool BeyondCompare
git config --local mergetool.path 'BeyondCompare安装路径'
git config --local mergetool.keepbackup false

    在Git中使用Beyond Compare解决冲突

    git mergetool
