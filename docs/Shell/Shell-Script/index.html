    Shell和Shell脚本 #

    Shell是一个命令行解释器，它的作用就是遵循一定的语法将输入的命令加以解释并传给系统。它为用户提供了一个向Linux发送请求以便运行程序的接口系统级程序，用户可以用Shell来启动、挂起、停止甚至是编写一些程序。 Shell本身是一个用C语言编写的程序，它是用户使用Linux的桥梁。Shell既是一种命令语言，又是一种程序设计语言。作为命令语言，它互动式地解释和执行用户输入的命令；作为程序设计语言，它定义了各种变量和参数，并提供了许多在高阶语言中才具有的控制结构，包括循环和分支。它虽然不是 Linux系统内核的一部分，但它调用了系统内核的大部分功能来执行程序、创建文档并以并行的方式协调各个程序的运行。

    Shell脚本是一种文本文件，其中包含了一系列的命令，可以通过解释器来执行。使用Shell脚本可以提高工作效率，减少重复性工作。

    Shell的优势 #

    虽然目前有很多脚本编程语言，但对于Linux操作系统而言，Shell是最好的工具，Linux底层命令都支持Shell语句，同时结合grep、sed和awk可以实现更高级的用法。Shell擅长系统管理脚本的开发，如软件启停脚本、监控报警脚本、日志分析脚本等。

    Shebang #

    在计算领域中，Shebang（也称为 Hashbang ）是一个由井号和叹号构成的字符序列 #! ，其出现在文本文件的第一行的前两个字符。 在文件中存在 Shebang 的情况下，类 Unix 操作系统的程序加载器会分析 Shebang 后的内容，将这些内容作为解释器指令，并调用该指令，并将载有 Shebang 的文件路径作为该解释器的参数。

    以#! /bin/sh开头的文件，程序在执行的时候会调用/bin/sh，也就是Bash解释器去执行文件中的代码。

    以#! /usr/bin/python开头的文件，程序在执行的时候会调用/usr/bin/python，代表指定使用Python解释器去执行文件中的代码。

    以#! /usr/bin/env开头的文件，是一种在不同平台上都能正确找到解释器的方法。

    注意事项：

    • 如果脚本文件指定了Shebang解释器，在脚本文件执行的时候，脚本文件名会作为参数传递给指定的解释器。
    • 如果脚本文件未指定Shebang解释器，在脚本文件执行的时候，默认会使用当前Shell（echo $SHELL）去解释执行脚本文件.
    • 如果脚本文件指定的Shebang解释器没有可执行权限，则会报错 “Bad interpreter：Permission denied”。
    • 如果脚本文件指定的Shebang解释器不存在，则会报错 “Bad interpreter：No such file or directory”。
    • 如果脚本文件指定的Shebang解释器不是一个可执行的文件，那么指定的解释器将会被忽略，转而交给当前的Shell去执行这个脚本。
    • 脚本文件指定的Shebang解释器需要使用绝对路径。
    • 如果在Shell中指定解释器执行文件，则文件指定的Shebang解释器将会被忽略。

    注释 #

    在Shell脚本中使用#进行注释，注释可以单独写一行，也可以跟在命令后面。示例如下：

    #! /bin/bash

# Description: 这是一个脚本文件
# Date: 2023-01-01
# Author: HC

echo "这是一个脚本文件！"

    变量 #

    定义变量 #

    Shell属于弱类型语言，无需声明变量类型，可以直接定义变量并使用。Shell中定义的变量，数据类型默认为字符串。

    需要注意的是，Shell在定义变量的时候，变量名和等号之间不能有空格。同时，变量名的命名须遵循如下规则：

    • 命名只能使用英文字母，数字和下划线，首字符不能以数字开头。
    • 中间不能有空格，可以使用下划线 _。
    • 不能使用标点符号。
    • 不能使用bash里的关键字（可用help命令查看保留关键字）。

    在Shell中定义变量和调用变量的示例如下：

    #! /bin/bash
# 定义变量name，并赋值为Shell
name="Shell"
# 通过echo调用并显示name，调用变量时需要使用 ${变量名称}的方式调用变量，{}可以省略，但不推荐。
echo ${name}

    在Shell中，变量赋值使用单引号与双引号是有区别的

    • 单引号''：不识别特殊语法。单引号里的任何字符都会原样输出，单引号字符串中的变量是无效的。
    • 双引号""：能够识别特殊语法。双引号里可以有变量，双引号里可以出现转义字符。
    • 反引号：能够执行反引号中的命令，并将命令执行的结果赋值给变量。等同于$()的用法。

    变量的类型 #

    Shell中的变量类型有如下几种：

    • 有的变量只能在函数内部使用，这叫做局部变量（local variable）。在 Shell 函数中定义的变量默认也是全局变量，它和在函数外部定义变量拥有一样的效果。要想变量的作用域仅限于函数内部，可以在定义时加上local命令，此时该变量就成了局部变量
    • 有的变量可以在当前 Shell 进程中使用，这叫做全局变量（global variable）。所谓全局变量，就是指变量在当前的整个 Shell 进程中都有效。每个 Shell 进程都有自己的作用域，彼此之间互不影响。在 Shell 中定义的变量，默认就是全局变量。
    • 而有的变量还可以在子进程中使用，这叫做环境变量（environment variable）。全局变量只在当前 Shell 进程中有效，对其它 Shell 进程和子进程都无效。如果使用export命令将全局变量导出，那么它就在所有的子进程中也有效了，这称为“环境变量”。

    全局变量和环境变量分又为内置环境变量和自定义环境变量。内置的特殊变量，如：$?的用途是返回上一条命令的执行结果，返回0表示成功，返回1-255表示不同类型的错误码。自定义变量，如：var=value表示给变量var赋值value。

    变量的作用域 #

    Shell 变量的作用域（Scope），就是 Shell 变量的有效范围（可以使用的范围）。

    通过pstree命令可以查看进程树，从而来判断当前shell是在那个作用域中。

    使用pstree命令需要安装psmisc，CentOS的安装命令为yum -y install psmisc

    使用bash和source执行脚本的区别：

    • 使用bash或者sh执行脚本时，会开启子Shell进程执行脚本。
    • 使用source或者点符号（.）执行脚本时，会在当前Shell进程中执行脚本。

    特殊变量 #

    Shell中有以下几个特殊变量：

    • $0：在Shell脚本中，获取Shell脚本文件名和脚本路径。
    • $n：在Shell脚本中，获取Shell脚本的第n个参数，n在1~9之间，如$1，$2，%3，n大于9 则需要写成${10}，参数之间需要使用空格隔开。
    • $#：在Shell脚本中，获取执行Shell脚本的参数个数。
    • $*：在Shell脚本中，获取执行Shell脚本的所有参数，不加引号等同于$@，加上引号"$*"作用是把接收所有参数作为一个字符串输出。
    • $@：在Shell脚本中，获取执行Shell脚本的所有参数，不加引号等同于$*，加上引号"$@"作用是把接收所有参数作为独立的字符串输出，空格保留。
    • $?：返回上一条命令的执行状态，返回值为0表示执行成功，非0表示执行失败。
    • $$：获取当前Shell脚本的进程号（PID）。
    • $!：获取上一个后台进程的进程号（PID）。
    • $_：获取上一条命令的最后一个参数。

    特殊变量的查询方式：

    执行man bash命令，搜索special Parameters。

    Shell字符串的操作 #

    Shell中常见的子串有以下几个：

    ${变量}：返回变量值。

    ${#变量}：计算变量返回值的长度。

    ${变量:start}：从变量返回值中索引为start位置开始取值。

    ${变量:start:length}：从变量返回值中索引为start位置开始取值，取值长度为length。

    ${变量#word}：从变量开头删除最短匹配的word。

    ${变量##word}：从变量开头删除最长匹配的word。

    ${变量%word}：从变量结尾删除最短匹配的word。

    ${变量%%word}：从变量结尾删除最长匹配的word。

    ${变量/pattern/string}：用string替代第一个匹配的pattern。

    ${变量//pattern/string}：用string替代全部匹配的pattern。

    Shell的条件测试（test） #

    test命令用于评估一个表达式的结果是真还是假。如果条件为真，那么命令执行状态码为0，如果条件为假，那么命令执行状态码为非0。

    test命令的语法共有四种：

    条件测试语法 说明
    test <测试表达式> 利用test命令进行条件测试表达式的方法，test命令和<测试表达式>之间至少有一个空格。
    [ <测试表达式> ] 通过单中括号进行条件测试的表达式的方法。单中括号和内容之间至少有一个空格。此方法使用最多
    [[ <测试表达式> ]] 通过双中括号进行条件测试的表达式的方法。新语法，双中括号和内容之间至少有一个空格。
    ((<测试表达式>)) 通过双小括号进行条件测试的表达式的方法。一般用于if语句里，双小括号两端不需要有空格。

    test命令关于文件名类型侦测的参数如下：

    语法示例：test -e filename

    参数 说明
    -e 判断该文件名是否存在（常用）
    -f 判断该文件名是否为文件（file）（常用）
    -d 判断该文件名是否为目录（directory）（常用）
    -b 判断该文件名是否为一个block device装置
    -c 判断该文件名是否为一个character device装置
    -s 判断该文件名是否为一个Socket文件
    -p 判断该文件名是否为一个FIFO（pipe）文件
    -L 判断该文件名是否为一个连结档

    test命令关于文件名类型侦测的参数如下：

    语法示例：test -r filename

    参数 说明
    -r 侦测该文件名是否具有可读的属性
    -w 侦测该文件名是否具有可写的属性
    -x 侦测该文件名是否具有可执行的属性
    -u 侦测该文件名是否具有SUID的属性
    -g 侦测该文件名是否具有SGID的属性
    -k 侦测该文件名是否具有Sticky bit的属性
    -s 侦测该文件名是否具有是否为非空白文件

    test命令关于两个文件之间比较的参数如下：

    语法示例：test file1 -nt file2

    参数 说明
    -nt newer than，判断file1是否比file2新
    -ot older than，判断file1是否比file2旧
    -ef 判断file1于file2是否为同一个文件

    test命令关于两个整数之间比较的参数如下：

    语法示例：test n1 -eq n2

    参数 说明
    -eq 两个数值相等（equal）
    -ne 两个数值不等（not equal）
    -gt n1大于n2（greater than）
    -lt n1小于n2（less than）
    -ge n1大于等于n2（greater than or equal）
    -le n1小于等于n2（less than or equal）

    test命令关于字符串判定的参数如下：

    语法示例：test -n string

    语法示例：test str1 = str2

    参数 说明
    -z 判断是否为空字符串，若为空，则返回true
    -o 判断是否为空字符串，若为空，则返回false
    = 判断字符串是否相等，若相等，则返回true
    != 判断字符串是否不相等，若不相等，则返回true

    test命令关于多重条件判断的参数如下：

    语法示例：test -r filename -a -x filename

    参数 说明
    -a （and）两个条件同时成立时才返回true
    -o （or）两个条件中任意一个成立则返回true
    ! 反向状态，如：test ! -x file，当file不具有-x属性时，则返回true

    通过单中括号进行条件测试的表达式的方法与利用test命令进行条件测试表达式的方法的作用相同，但在日常脚本的编写中，使用最多的是通过单中括号进行条件测试。

    语法示例：[ -e "$变量名称" ]

    使用单中括号进行条件测试需要注意两点：

    • 单中括号和内容之间必须至少有一个空格。
    • 在条件测试中使用变量，必须添加双引号。

    Shell流程控制 #

    if 判断语句 #

    单分支if条件判断语法格式：

    if <条件表达式>; then
    ......
    <代码部分>
    ......
fi

    if <条件表达式>; then
    ......
    <代码部分>
    ......
else
    ......
    <代码部分>
    ......
fi

    多分支if条件判断语法格式：

    if <条件表达式1>; then
    ......
    <代码部分>
    ......
elif <条件表达式2>; then
    ......
    <代码部分>
    ......
else
    ......
    <代码部分>
    ......
fi

    if和elif的条件表达式后面需要使用then，else后面不需要。

    for 循环 #

    语法格式：

    for var in item1 item2 ... itemN
do
    command1
    command2
    ...
    commandN
done

    while 循环 #

    语法格式：

    while condition
do
    command
done

    无限循环：

    while true
do
    command
done

    until 循环 #

    until 循环执行一系列命令直至条件为 true 时停止。

    until 循环与 while 循环在处理方式上刚好相反。

    一般 while 循环优于 until 循环，但在某些时候—也只是极少数情况下，until 循环更加有用。

    until condition
do
    command
done

    condition 一般为条件表达式，如果返回值为 false，则继续执行循环体内的语句，否则跳出循环。

    case 选择语句 #

    语法

    case 值 in
条件1 )
    command1
    command2
    ...
    commandN
    ;;
条件2 )
    command1
    command2
    ...
    commandN
    ;;
esac

    示例：

    read -p "请输入 1 到 4 之间的数字:" Num
case $Num in
    1)  echo '你选择了 1'
    ;;
    2)  echo '你选择了 2'
    ;;
    3)  echo '你选择了 3'
    ;;
    4)  echo '你选择了 4'
    ;;
    *)  echo '你没有输入 1 到 4 之间的数字'
    ;;
esac

    跳出循环 #

    在循环过程中，有时候需要在未达到循环结束条件时强制跳出循环，Shell 使用两个命令来实现该功能：break 和 continue。

    break 命令 #

    break 命令允许跳出所有循环（终止执行后面的所有循环）。

    continue #

    continue 命令与 break 命令类似，只有一点差别，它不会跳出所有循环，仅仅跳出当前循环。

    Shell 函数 #

    语法格式：

    function 函数名称() {
    ......
    函数体
    ......
}

    使用函数名称可以直接调用函数

    Shell脚本示例 # 

    #! /bin/bash
# 作者：HC
# 创建时间：2023年8月15日
# 脚本描述：此脚本用于对网站可用性的监测

# 定义网站URL
URL=http://10.33.11.48
# 定义超时时间
timeout=3
# 定义访问失败计数器
fail_tag=0
# 定义邮件收件人
recipient="HC@mail.com"

# 定义检测网站URL的函数
function CheckURL(){
    while true; do
        # 通过wget命令检测网站的可用性
        wget --timeout=${timeout} --tries=1 ${URL} -q -O /dev/null
        # 条件判断，如果命令执行结果不等于0，
        if [ $? -ne 0 ]; then
            (( fail_tag+=1 ))
        fi
        # 判断访问失败次数大于三次，则报警发送邮件
        if [ ${fail_tag} -gt 3 ]; then
            echo "此邮件来自于自动巡检系统，"$URL"目前无法访问，请尽快前往确认！" | mail -s "网站故障通知！" ${recipient}
            exit 2
        else
            # 如果网站正常访问，则3秒后继续循环
            sleep 3
        fi
    done
}

# 调用检测网站函数
