    Prometheus简介 #

    官方网站：https://prometheus.io

    Prometheus 是一个开源系统，最初是在 SoundCloud 上构建的监控和警报工具包。自2012年成立以来，许多公司和组织都采用了Prometheus，该项目具有非常活跃的开发人员和用户 社区。它现在是一个独立的开源项目，并独立于任何公司进行维护。强调这一点，并澄清该项目的治理结构，Prometheus于2016年加入 云原生计算基金会 作为继 Kubernetes 之后的第二个托管项目。

    Prometheus 收集其指标并将其存储为时间序列数据，即指标信息与记录它的时间戳一起存储，以及称为标签的可选键值对。

    Prometheus安装 #

    创建软件安装目录

    mkdir -p /opt/installed

    下载 Prometheus 2.37.5 长期支持版

    wget -P /opt/installed https://github.com/prometheus/prometheus/releases/download/v2.37.5/prometheus-2.37.5.linux-amd64.tar.gz

    解压缩安装包

    tar zxvf prometheus-2.37.5.linux-amd64.tar.gz /opt

    创建软连接

    ln -s /opt/prometheus-2.37.5.linux-amd64 /usr/local/prometheus

    Prometheus启动 #

    快速启动 #

    进入Prometheus目录，执行Prometheus程序就可以直接启动Prometheus

    nohup /usr/local/prometheus/prometheus --config.file=/usr/local/prometheus/prometheus.yml >/dev/null 2>&1 &

    生产环境启动 #

    创建并编辑/usr/lib/systemd/system/prometheus.service文件

    [Unit]
Description=Prometheus
After=network.target

[Service]
Type=simple
ExecStart=/usr/local/prometheus/prometheus --config.file=/usr/local/prometheus/prometheus.yml

[Install]
WantedBy=multi-user.target

    重启载入配置文件

    systemctl daemon-reload

    配置 Prometheus 随系统启动

    systemctl enable prometheus.service

    启动 Prometheus 服务

    systemctl start prometheus.service

    常用启动参数介绍 #

    --config.file="prometheus.yml"：指定配置文件，默认在安装目录下。

    --web.read-timeout=5m：请求链接的最大等待时间，默认5分钟。

    --web.max-connections=512：最大同时链接数，默认512。

    --srorage.tsdb.path="data/"：指定采集数据的存储路径，默认存储在/data。

    --storage.tsdb.retention.time=30d：指定采样在存储中的保留时间，一般设置为15至30天为宜。

    --query.timeout=2m：指定查询超时时间，默认2分钟。

    --query.max-concurrency=20：指定查询的最大并发数，默认20个。

    Prometheus数据采集方式 #

    Pull方式采集数据（主动） #

    Pull方式采集数据是指先在客户端（被监控机）上安装各类 exporters ，并以守护进程的方式运行。 exporters 本身也是一个HTTP_Server，可以对HTTP请求作出响应和返回请求数据。Prometheus使用Pull方式采集数据，其实就是通过HTTP请求的方式去获取每个节点上 exporters 所采集的数据。

    Push方式采集数据（被动） #

    Push方式采集数据是指在任意节点（可以是服务端、客户端或独立节点）安装 PushGateway 插件，通过自己开发的各种监控脚本把监控数据组织成 Key / Value 的形式发送给 PushGateway，然后再由 PushGateway 发送给Prometheus服务。

    METRIC类型 #

    官网链接：https://prometheus.io/docs/concepts/metric_types/#metric-types

    Counter（计数器） #

    Counter（计数器）是一个累积型的指标，其值在重新启动时只能增加或重置为零。例如，可以使用计数器来表示已送达的请求数量、已完成的任务数量或错误数量。

    不要使用Counter来暴露可以减少的值。例如，不要对当前正在运行的进程数量使用Counter；而是使用Gauge（仪表）。

    计数器的客户端库使用文档：

    Gauge（测量） #

    Gauge（测量）是表示可以任意上下起伏数值的指标，其返回值是一个瞬时状态。

    Gauge常用于测量值，如温度或当前内存使用情况，但也用于可以上下变动的“计数”，如并发请求的数量。

    Histogram（直方图） #

    Histogram（直方图）对观测结果进行采样（通常如请求持续时间或响应大小），并在可配置的buckets中计数。它还提供了所有观察到的值的总和。

    Histogram的基本度量名称为<basename>，在刮擦期间暴露了多个时间序列：

    观察buckets的累积计数器，暴露为<basename>_bucket{le="<upper inclusive bound>"}

    所有观察到的值的总和，公开为<basename>_sum

    已观察到的事件计数，暴露为<basename>_count（与上面的<basename>_bucket{le="+Inf"}相同）

    使用 histogram_quantile() 函数从直方图甚至直方图聚合中计算分位数。直方图也适合计算Apdex分数。在存储桶上操作时，请记住直方图是累积的。有关直方图使用情况和摘要差异的详细信息，请参阅直方图和摘要。

    Histograms的客户端库使用文档:

    • Go
    • Java
    • Python
    • Ruby
    • .Net

    Summary（摘要） #

    与Histogram（直方图）类似，Summary（摘要）样本观察（通常是请求持续时间和响应大小）。虽然它还提供了观测的总计数和所有观测值的总和，但它在滑动时间窗口上计算可配置的分位数。

    具有<basename>基本度量名称的摘要在抓取期间暴露了多个时间序列：

    流式φ-quantiles（0 ≤ φ ≤ 1）观察到的事件，暴露为<basename>{quantile="<φ>"}

    所有观察到的值的总和，公开为<basename>_sum

    已观察到的事件计数，暴露为<basename>_count

    有关φ分位数、摘要用法和与直方图的差异的详细解释，请参阅直方图和摘要。

    # my global config
global:
  scrape_interval: 15s   # 全局默认的数据拉取间隔
  evaluation_interval: 15s   # 全局默认的规则(主要是报警规则)拉取间隔
 
# Alertmanager configuration
alerting:
  alertmanagers:
  - static_configs:
    - targets:
      # - alertmanager:9093
 
# Load rules once and periodically evaluate them according to the global 'evaluation_interval'.
rule_files:
  # - "first_rules.yml"
  # - "second_rules.yml"
 
# A scrape configuration containing exactly one endpoint to scrape:
# Here it's Prometheus itself.
scrape_configs:
  # The job name is added as a label `job=` to any timeseries scraped from this config.
  - job_name: 'prometheus'
 
    # metrics_path defaults to '/metrics'

    Prometheus常用函数 #

    官网链接：https://prometheus.io/docs/prometheus/latest/querying/functions/

    rate函数 #

    rate()函数是专门搭配Counter（计数器）类型数据来使用的。它的功能是计算出一个时间段内平均每秒的增量。

    示例：

    在Prometheus中获取1分钟内每秒网络的接收字节数

    rate(node_network_receive_bytes_total[1m])

    increase函数 #

    increase()函数也是专门搭配Counter（计数器）类型数据来使用的。它的功能是计算出一段时间内增加的总量。

    示例：

    在Prometheus中获取1分钟内网络的接收字节数

    increase(node_network_receive_bytes_total[1m])

    经验：在遇到Counter（计数器）类型数据的时候首先要做的就是套上rate()函数或者instance()函数。

    sum函数 #

    sum()函数会把所有监控节点结果集进行求和，如果需要分组的话，则需要通过by()来实现。

    示例：

    对所有监控节点的网络接收字节数束进行求和

    sum(rate(node_network_receive_bytes_total[1m]))

    对所有监控节点的网络接收字节数进行分组求和

    sum(rate(node_network_receive_bytes_total[1m])) by (instance)

    topk函数 #

    topk()函数可以实现取前几位最高值的功能。

    示例：

    取所有监控节点中网络接收字节数前3的节点

    topk(3,rate(node_network_receive_bytes_total[1m]))

    topk()函数在实际应用中一般不会用来观测数据，而是用来进行瞬时报警。

    count函数 #

    count()函数用于计算出符合条件的节点数量。

    示例：

    count(rate(node_cpu_seconds_total[1m]) > 0.5)

    count()函数一般用于监控判断，例如，一个集群中有100台节点，当CPU使用率达到80%的节点数量达到50台的时候，就需要触发报警。

    常用监控指标的计算公式 #

    系统信息 #

    获取系统信息

    node_uname_info{job="node_exporter"}

    系统运行时间

    sum(time() - node_boot_time_seconds{job="node_exporter"}) by (instance)

    time() ：系统当前时间

    node_boot_time_seconds：系统启动时间

    CPU #

    CPU核数

    count(node_cpu_seconds_total{job="node_exporter",mode='system'}) by (instance)

    CPU使用率

    (1 - ((sum(increase(node_cpu_seconds_total{mode="idle"}[1m])) by (instance)) / (sum(increase(node_cpu_seconds_total[1m])) by (instance))))) * 100

    CPU空闲模式占比

    内存 #

    内存使用率 = （总内存 - 空闲内存 + 缓冲缓存 + 页面缓存） / 总内存 * 100

    (node_memory_MemTotal_bytes - node_memory_MemFree_bytes - node_memory_Buffers_bytes - node_memory_Cached_bytes ) / node_memory_MemTotal_bytes * 100

    node_memory_MemTotal_bytes：总内存大小 node_memory_MemFree_bytes：空闲内存大小 node_memory_Buffers_bytes：缓冲缓存大小 node_memory_Cached_bytes：页面缓存大小
